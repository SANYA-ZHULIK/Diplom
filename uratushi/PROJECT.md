## Функциональные модули

### 1. Аутентификация (auth.js)
- Регистрация нового пользователя
- Вход в систему
- Выход
- Проверка сессии при загрузке
- Автоматическое создание профиля

### 2. Карта зала (hall-map.js)
- Загрузка столиков из БД
- Отображение статусов цветом:
  - 🟢 Зелёный - свободен
  - 🟡 Жёлтый - ожидает подтверждения
  - 🔴 Красный - занят
  - ⚪ Серый - на ремонте
- Автообновление каждые 30 секунд
- Realtime подписки на изменения

### 3. Бронирование (booking.js)
- Создание бронирования
- Проверка доступности перед бронированием
- Отмена бронирования (>2 часов до визита)
- Валидация данных

### 4. Личный кабинет (profile.js)
- Информация о пользователе
- Список предстоящих бронирований
- История бронирований
- Баланс бонусных баллов
- История начислений/списаний

### 5. Администрирование (admin.js)
- Просмотр всех бронирований
- Фильтры по дате и статусу
- Подтверждение/отклонение заявок
- Отметка визитов (начисление 10 баллов)
- CRUD операции со столиками
- Ручное управление бонусами клиентов

## База данных Supabase

### Таблицы

1. **tables** - Столики ресторана
   - id (PK)
   - table_number
   - seats
   - position_x, position_y
   - status (active/maintenance)

2. **bookings** - Бронирования
   - id (PK)
   - user_id (FK → auth.users)
   - table_id (FK → tables)
   - booking_date, booking_time, end_time
   - duration
   - guest_name, guest_phone
   - notes
   - status (pending/confirmed/completed/cancelled)

3. **profiles** - Профили пользователей
   - id (PK/FK → auth.users)
   - full_name, phone
   - role (user/admin)
   - bonus_balance
   - created_at, updated_at

4. **bonus_history** - История бонусов
   - id (PK)
   - user_id (FK → auth.users)
   - amount
   - reason
   - created_at

### Безопасность (RLS)

Включены политики безопасности на уровне строк:
- Пользователи видят только свои данные
- Админы имеют доступ ко всем данным
- Изменения данных защищены от несанкционированного доступа

## Особенности реализации

### Realtime обновления
- Карта зала обновляется автоматически при изменении статуса столика
- Список бронирований синхронизируется между клиентом и админом
- Нет необходимости перезагружать страницу

### Адаптивность
- Полная поддержка мобильных устройств (320px+)
- Адаптивная навигация
- Оптимизированные таблицы для маленьких экранов

### UX улучшения
- Плавные анимации
- Toast-уведомления вместо алертов
- Валидация форм с моментальной обратной связью
- Заглушки для пустых состояний
- Индикаторы загрузки

### Производительность
- Минимальное количество запросов к БД
- Оптимизированные индексы
- Кэширование данных на клиенте
- Async/await для неблокирующих операций

## Процесс деплоя

1. **Настройка Supabase** (10 минут)
   - Создание проекта
   - Выполнение SQL-схемы
   - Получение API ключей

2. **GitHub Pages** (5 минут)
   - Создание репозитория
   - Загрузка файлов
   - Активация Pages

3. **Конфигурация** (5 минут)
   - Обновление SUPABASE_URL и SUPABASE_KEY
   - Настройка аутентификации
   - Назначение администратора

**Общее время развёртывания: ~20 минут**
