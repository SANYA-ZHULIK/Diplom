# 🚀 Инструкция по развертыванию приложения "У Ратуши"

## Шаг 1: Настройка Supabase

### 1.1 Создание проекта
1. Перейдите на [supabase.com](https://supabase.com) и войдите в аккаунт
2. Нажмите **New Project**
3. Заполните данные:
   - **Name**: uratushi
   - **Database Password**: создайте надежный пароль
   - **Region**: выберите ближайший (например, West Europe)
4. Нажмите **Create new project** (создание займет 1-2 минуты)

### 1.2 Загрузка схемы базы данных
1. В панели проекта выберите **SQL Editor** (в левом меню)
2. Нажмите **New query**
3. Скопируйте содержимое файла `supabase-schema.sql` из репозитория
4. Вставьте в редактор и нажмите **Run**

### 1.3 Настройка аутентификации
1. Перейдите в **Authentication** → **Settings**
2. В разделе **Site URL** укажите URL вашего приложения (пока можно `http://localhost`)
3. Сохраните изменения

### 1.4 Получение API ключей
1. Перейдите в **Settings** → **API**
2. Скопируйте:
   - **Project URL** (например: `https://xbwfrfnagesceeozazcr.supabase.co`)
   - **anon/public** ключ (начинается с `eyJ...`)

### 1.5 Назначение администратора
1. Зарегистрируйтесь в приложении обычным способом
2. В SQL Editor выполните:
```sql
-- Замените email на ваш
UPDATE profiles SET role = 'admin' WHERE id IN (
    SELECT id FROM auth.users WHERE email = 'ваш-email@example.com'
);
```

## Шаг 2: Настройка GitHub Pages

### 2.1 Создание репозитория
1. Перейдите на [github.com](https://github.com)
2. Нажмите **New repository**
3. Название: **uratushi**
4. Сделайте репозиторий **Public**
5. Нажмите **Create repository**

### 2.2 Загрузка файлов
```bash
# Если используете терминал
cd uratushi
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ВАШ-USERNAME/uratushi.git
git push -u origin main
```

Или загрузите файлы через веб-интерфейс:
1. Откройте созданный репозиторий
2. Нажмите **uploading an existing file**
3. Выберите все файлы из папки `uratushi/`
4. Нажмите **Commit changes**

### 2.3 Включение GitHub Pages
1. В репозитории перейдите в **Settings** → **Pages**
2. В разделе **Source** выберите:
   - **Branch**: `main`
   - **Folder**: `/ (root)`
3. Нажмите **Save**

Через несколько минут сайт будет доступен по адресу:
```
https://ваш-username.github.io/uratushi/
```

## Шаг 3: Обновление Supabase URL

### 3.1 Обновление настроек в коде
В файле `js/supabase.js` замените значения:

```javascript
const SUPABASE_URL = 'https://ваш-project-id.supabase.co';
const SUPABASE_KEY = 'ваш-anon-key';
```

### 3.2 Обновление URL в Supabase
1. В Supabase перейдите в **Authentication** → **Settings**
2. В **Site URL** укажите ваш GitHub Pages URL
3. В **Redirect URLs** добавьте:
   - `https://ваш-username.github.io/uratushi/`
   - `https://ваш-username.github.io/uratushi/index.html`
4. Сохраните изменения

## 🔧 Устранение неполадок

### Проблема: "Ошибка загрузки карты"
**Решение**: Проверьте, что таблица `tables` содержит данные:
```sql
SELECT * FROM tables;
```

### Проблема: "Не удается войти"
**Решение**: 
1. Проверьте правильность SUPABASE_URL и SUPABASE_KEY
2. Убедитесь, что в Supabase Authentication настроены разрешенные URL
3. Проверьте консоль браузера (F12) на ошибки CORS

### Проблема: "Realtime не работает"
**Решение**: В Supabase Database → Replication включите Realtime для таблиц:
- `bookings`
- `tables`

### Проблема: "Админ-панель не доступна"
**Решение**: Убедитесь, что выполнили SQL-запрос для назначения роли admin
